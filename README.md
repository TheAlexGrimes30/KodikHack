# Kodik Project — Система калиброванных ставок

## Описание проекта

Система калиброванных ставок — интеллектуальная платформа поддержки принятия решений при запуске новых продуктов, фич или бизнес-инициатив в условиях неопределённости.

Основная идея проекта — заменить одно большое рискованное решение серией небольших проверяемых ставок. Каждая ставка проходит анализ, выполняется на практике и после получения фактических результатов влияет на доверие к следующим решениям.

Архитектура проекта основана на пяти основных компонентах:

1. **Намерение (Intent)** — описание идеи и извлечение скрытых допущений.
2. **Среда (Environment)** — моделирование внешних факторов и сценариев.
3. **Деструктор (Attack)** — поиск наиболее вероятных причин провала.
4. **Калибровка (Calibration)** — сравнение прогноза с реальным результатом.
5. **Порог (Threshold)** — принятие решения о следующем шаге.

# Архитектура БД

Главной сущностью системы является **Session** — один полный цикл анализа одной ставки.

```
User
   │
   ▼
Project
   │
   ▼
Session
   ├──────────────┐
   ▼              ▼
Intent        EnvScenario
   │              │
   ▼              ▼
Assumption     Attack
        │
        ▼
ThresholdDecision
        │
        ▼
Bet
        │
        ▼
CalibrationRecord
```

Дополнительно используются системные таблицы:

- Configurator
- IndustryBenchmark
- FailureCase
- DataFeed
- MacroIndicator
- EnvironmentEvent
- SessionLog

# Описание таблиц

## users

Пользователи системы.

| Поле | Назначение |
|------|------------|
| id | UUID пользователя |
| email | Email |
| display_name | Отображаемое имя |

---

## projects

Проекты пользователя.

Хранят информацию о продукте или стартапе.

---

## sessions

Одна аналитическая сессия.

Объединяет результаты работы всех агентов.

---

## intents

Описание намерения пользователя.

Содержит:

- сумму инвестиций;
- горизонт;
- ожидаемый результат;
- бизнес-метрики.

---

## assumptions

Допущения, на которых строится идея.

Каждое допущение анализируется отдельно.

---

## env_scenarios

Сценарии внешней среды.

Например:

- рост ставки ЦБ;
- появление конкурента;
- падение спроса.

---

## attacks

Результаты работы Деструктора.

Каждая запись показывает:

- какое допущение атакуется;
- каким сценарием;
- возможные последствия.

---

## threshold_decisions

Финальное решение системы.

Возможные вердикты:

- Scale
- Probe
- Hold
- Exit

Также хранится уровень доверия (trust level).

---

## bets

История всех ставок.

Содержит:

- прогноз;
- реальные результаты;
- фактические затраты.

---

## calibration_records

Результаты калибровки.

Содержит:

- абсолютную ошибку;
- относительную ошибку;
- optimism bias;
- изменение доверия.

---

## configurators

Конфигурации различных вертикалей проекта.

Например:

- IT Startup;
- FinTech;
- LegalTech.

---

## industry_benchmarks

Отраслевые показатели.

Используются агентом Среды.

---

## failure_cases

Исторические кейсы неудачных проектов.

Используются Деструктором.

---

## data_feeds

Источники внешних данных.

Например:

- Центральный Банк;
- Росстат;
- новости.

---

## macro_indicators

Макроэкономические показатели.

Например:

- инфляция;
- ключевая ставка;
- курс валют.

---

## environment_events

Внешние события.

Например:

- санкции;
- кризисы;
- изменения законодательства.

---

## session_logs

Логи работы агентов.

Используются для аудита.

# Запуск проекта

## 1. Клонирование

```bash
git clone <repository_url>

cd KodikProject
```

---

## 2. Создание виртуального окружения

Windows

```bash
python -m venv .venv
```

Linux

```bash
python3 -m venv .venv
```

---

## 3. Активация

Windows

```bash
.venv\Scripts\activate
```

Linux

```bash
source .venv/bin/activate
```

---

## 4. Установка зависимостей

```bash
pip install -r requirements.txt
```

---

## 5. Настройка переменных окружения

Создать файл `.env`

```env
DATABASE_URL=postgresql+psycopg://postgres:password@localhost:5432/kodik
```

---

## 6. Создание миграции

```bash
alembic revision --autogenerate -m "Initial schema"
```

---

## 7. Применение миграций

```bash
alembic upgrade head
```

---

## 8. Запуск сервера

```bash
uvicorn backend.main:app --reload
```

Swagger:

```
http://localhost:8000/docs
```

# Полезные команды Alembic

Создать миграцию

```bash
alembic revision --autogenerate -m "message"
```

Применить миграции

```bash
alembic upgrade head
```

Откатить последнюю

```bash
alembic downgrade -1
```

Откатить все

```bash
alembic downgrade base
```

Текущая ревизия

```bash
alembic current
```

История

```bash
alembic history
```
